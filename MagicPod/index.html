About

一言でいうと

MagicPodは、Webアプリやモバイルアプリの自動テストを、コードを書かずに作成・実行できるテスト自動化ツールです。

PlaywrightやSeleniumではコードを書いてテストを作成しますが、MagicPodでは画面上で操作を組み合わせてテストを作成します。

GUI

GUIとは？

GUI（Graphical User Interface）とは、画面を見ながらマウスで操作できる仕組みのことです。

Click

ボタンをクリック

画面上のボタンやリンクを選んで操作します。

Input

テキストを入力

フォームや検索欄に文字を入力します。

Select

メニューを選択

プルダウンやメニューから項目を選びます。

Drag

ドラッグ＆ドロップ

画面上の要素をつかんで移動します。

MagicPodでは、このGUIを使ってテストを作成します。

Difference

Playwrightとの違い

同じログインテストでも、Playwrightはコードで書き、MagicPodは画面上で手順を並べます。

Playwright

コードを書いてテストを作成

await page.goto("https://example.com");
await page.fill("#user", "hanako");
await page.click("#login");
MagicPod

画面上で手順を並べる

  1. URLを開く
  2. ユーザーIDを入力
  3. パスワードを入力
  4. ログインボタンをクリック
  5. ログイン成功を確認
コードを書く代わりに、GUIで操作を組み立てるのがMagicPodです。

Strength

MagicPodの特徴

Self-healing

軽微な画面変更に強い

例えばHTMLのIDが変わると、通常のSeleniumでは要素が見つからずにエラーになることがあります。 MagicPodはボタンの文字、HTML構造、周囲の要素、見た目なども参考にして対象を探します。

変更前

<button id="login">

変更後

<button id="btnLogin">

Limits

MagicPodが苦手なこと

複雑なアルゴリズム

機械学習、高度な数学計算、複雑なグラフ探索などには向いていません。

非常に複雑なループ処理

1000件比較、特殊な条件判定、さらに複雑な計算を組み合わせる処理は苦手です。

Playwrightほど自由度は高くない

PlaywrightではNode.jsが利用できるため、ほぼ何でも実装できます。MagicPodはGUI中心です。

Windowsデスクトップアプリの操作

Excel、メモ帳、業務Windowsアプリなどを自由に操作する用途には向いていません。

AutoItのようなWindows操作

座標クリック、キーボード操作、Windows全体の自動操作などは得意ではありません。
