Page Object

Page Object パターン

何をするもの？

画面操作をまとめて管理する設計方法

ボタンをクリックする処理や文字を入力する処理を、画面ごとにまとめます。

Page Objectを使わない場合

page.fill("#username", "test")
page.fill("#password", "pass")
page.click("#login-button")

ログインするたびに同じコードを書くことになります。

Page Objectを使う場合

login_page.login("test", "pass")

内部では次の処理を実行します。

page.fill("#username", "test")
page.fill("#password", "pass")
page.click("#login-button")

コードの重複を減らせる

同じ操作を何度も書かなくて済みます。

保守しやすい

ログインボタンのIDが変わった場合も、Page Object内だけ修正すれば済みます。

"#login-button"
"#submit-button"

役割：画面操作を管理する

Files

Page Object のファイル構成

画面ごとにファイルを分けて管理することが多いです。

プロジェクト
├─ test_main.py
└─ pages
   ├─ form.py
   └─ confirm.py

各ファイルの役割

ファイル 役割
test_main.py 何をテストするかを書く
form.py 入力画面の操作を書く
confirm.py 確認画面の操作・確認処理を書く

test_main.py

テストの流れを書く場所です。実際のクリックや入力処理は書かず、「何をしたいか」を中心に書きます。

form_page.open()

form_page.input_name("山田太郎")

form_page.click_confirm()

confirm_page.check_name("山田太郎")

form.py

入力画面の操作をまとめる場所です。

class FormPage:

    def input_name(self, name):
        self.page.fill("#username", name)

    def click_confirm(self):
        self.page.click("#submit")

文字入力、ボタンクリック、チェックボックス選択、プルダウン選択などを書きます。

confirm.py

確認画面の操作や確認処理をまとめる場所です。

class ConfirmPage:

    def check_name(self, expected):
        expect(
            self.page.locator("#username")
        ).to_have_text(expected)

全体の流れ

test_reserve.py
    ↓
FormPage
    ↓
ConfirmPage
ファイル 一言でいうと
test_reserve.py テストの流れを書く
form.py 入力操作を書く
confirm.py 確認処理を書く
Page Object パターンでは、「何をするか」と「どう操作するか」を分けることで、コードを読みやすく・修正しやすくします。

Data Driven

データ駆動テスト

テストデータを外部ファイルや変数などで管理し、同じテストを複数のデータで実行するテスト手法です。

何をするもの？

同じテストを複数のデータで実行する方法

テストケースを増やしやすい

コードを増やさずにテストパターンを追加できます。

テストデータを管理しやすい

CSVやExcelで管理することもできます。

役割：テストデータを管理する

test_data_driven.py
│
├─ utilities
│   └─ read_csv.py
│
├─ pages
│   ├─ form.py
│   └─ confirm.py
│
└─ datadriven
    └─ reservation.csv

read_csv.py

CSVを読む共通関数を書きます。

import csv

def read_csv_data(csv_path):
    rows = []
    with open(str(csv_path), encoding="utf-8") as csv_data:
        content = csv.reader(csv_data)
        next(content, None)
        for row in content:
            rows.append(row)
    return rows

test_data_driven.py

関数をインポートします。

from utilities.read_csv import read_csv_data

そしてCSVを読み込みます。

class TestDataDriven:
    datalist = read_csv_data(
        "datadriven/reservation.csv"
    )
reservation.csv
↓
read_csv_data()
↓
datalist に格納

役割の整理

  • read_csv.py：CSVを読む担当
  • test_data_driven.py：どのCSVを使うか決める担当
  • form.py：入力担当
  • confirm.py：確認担当

reservation.csv の例

name,term
山田太郎,3
鈴木花子,2

datalist の中身

[
    ["山田太郎", "3"],
    ["鈴木花子", "2"]
]

pytest

テストメソッドの引数として渡す処理

@pytest.mark.parametrize(
    "変数名1,変数名2,変数名3",
    datalist
)
def 関数名(self, page: Page, 変数名1, 変数名2, 変数名3):
    # テスト処理を書く

ポイント

  • @pytest.mark.parametrize() の変数名と、テストメソッドの引数名は同じにする
  • datalist の1行分のデータが順番に各変数へ代入される
  • CSVのヘッダー名と同じ変数名にすると分かりやすい

違いと関係性

違い

項目 Page Object パターン データ駆動テスト
管理するもの 画面操作 テストデータ
目的 コードを整理する テストケースを増やしやすくする
まとめるもの ボタン操作、入力処理など ユーザー名、パスワードなど

よく使う場所 画面操作部分 テストケース部分
