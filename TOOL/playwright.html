About

Playwrightとは

Playwright（プレイライト）は、Webブラウザを自動で操作できるツールです。

人がブラウザで行う操作（クリック、文字入力、画面遷移など）をプログラムで自動実行できます。

主な用途

  • Webアプリケーションのテスト
  • 繰り返し作業の自動化
  • 動作確認の効率化

Scope

できること・できないこと

できること

ブラウザ操作を自動化する

ブラウザの基本操作

  • クリック
  • 文字入力
  • チェックボックス選択
  • プルダウン選択
  • ページ遷移

画面内容の取得

  • 文字列の取得
  • ページタイトルの取得
  • URLの取得

その他

  • 自動的な待機処理
  • スクリーンショットの取得
  • 動画の取得
  • 実際のブラウザ操作をスクリプトに変換（Codegen）
  • Chrome、Edge、Firefox、Safariでの実行

「画面に○○が表示されるまで待つ」「ボタンが押せる状態になるまで待つ」といった待機処理も扱えます。

できないこと

ブラウザ以外は基本的に対象外

テストスクリプトの自動修復

画面変更によってテストが失敗した場合、セレクタの変更をPlaywrightが自動で直すことは基本的にできません。

ボタン名が変わった セレクタが変わった 自動で修正

ブラウザ外の操作

Playwrightは主にブラウザ操作用のツールです。以下のような操作は対象外です。

  • Excel操作
  • メモ帳操作
  • Windowsアプリ操作
  • ファイルエクスプローラー操作

Syntax

Playwrightの基本文法

pytestでPlaywrightを書くときの基本形です。

テストコードのひな型

from playwright.sync_api import Page
import pytest

class テスト名(object):

    @pytest.fixture(scope="function", autouse=True)
    def page_fixture(self, page: Page):
        self.page = page
        yield
        self.page.close()

    def test_change_all_params(self):
        page = self.page
        page.goto("URLをここに書く", wait_until="networkidle")

        # 操作内容をここに書いていく

        # キーボード操作
        page.press("セレクタ", "キー")

        # 文字入力
        page.fill("セレクタ", "入力したい文字")

        # クリック
        page.click("セレクタ")

        # チェックボックスを選択
        page.check("セレクタ")

        # プルダウンを選択
        page.select_option("セレクタ", "値")

        # スクリーンショットを保存
        page.screenshot(path="保存先")

        # Webページを開く
        page.goto("URL")

        # 現在のURLを取得
        page.url

        # ページタイトルを取得
        page.title()

        # テキストを取得
        page.text_content("セレクタ")

Assert

Playwrightのアサート

Playwrightでは主に、Python標準の assert と Playwright標準の expect で結果を確認します。

assert

Python標準

値を取得して、期待値と比較します。

expect

Playwright標準

要素の状態を確認します。表示されるまで自動で待機してくれるため、Playwrightではよく使われます。

Python Standard

assert

Python標準のアサートです。

完全一致

assert 実際の値 == 期待値

例

assert page.url == "https://example.com"

今開いているページのURLと期待値のURLを比較します。

部分一致

assert "文字列" in 実際の値

例

message = page.text_content("#message")

assert "登録完了" in message

URL確認

assert 変数名.url == "期待値URL"

例

assert page.url == "https://example.com/mypage"

タイトル確認

assert 変数名.title() == "期待値タイトル"

例

assert page.title() == "マイページ"

チェック状態確認

assert 変数名.is_checked("セレクタ")

例

assert page.is_checked("#agree")

表示確認

assert 変数名.is_visible("セレクタ")

例

assert page.is_visible("#message")

Playwright Standard

expect

Playwright標準のアサートです。使用する場合はインポートが必要です。

from playwright.sync_api import expect

URL確認

expect(変数名).to_have_url("期待値URL")

例

expect(page).to_have_url(
    "https://example.com/mypage"
)

タイトル確認

expect(変数名).to_have_title("期待値タイトル")

例

expect(page).to_have_title("マイページ")

テキスト完全一致

expect(ロケータ).to_have_text("期待値")

ロケータ（Locator）：画面上のボタンやテキストボックスなど、操作・確認したい要素を指定するためのものです。

例

expect(
    page.locator("#message")
).to_have_text("登録完了")

テキスト部分一致

expect(ロケータ).to_contain_text("文字列")

例

expect(
    page.locator("#message")
).to_contain_text("登録完了")

チェック状態確認

expect(ロケータ).to_be_checked()

例

expect(
    page.locator("#agree")
).to_be_checked()

表示確認

expect(ロケータ).to_be_visible()

例

expect(
    page.locator("#message")
).to_be_visible()

非表示確認

expect(ロケータ).not_to_be_visible()

例

expect(
    page.locator("#loading")
).not_to_be_visible()

assert と expect の違い

確認内容 assert expect
URL確認 assert page.url == "URL" expect(page).to_have_url("URL")
タイトル確認 assert page.title() == "タイトル" expect(page).to_have_title("タイトル")
テキスト完全一致 assert 実際の値 == "文字列" expect(locator).to_have_text("文字列")
テキスト部分一致 assert "文字列" in 実際の値 expect(locator).to_contain_text("文字列")
チェック状態確認 assert page.is_checked("セレクタ") expect(locator).to_be_checked()
表示確認 assert page.is_visible("セレクタ") expect(locator).to_be_visible()

assert

  • Python標準
  • 値を取得して比較する

expect

  • Playwright標準
  • 要素が表示されるまで自動で待機してくれる
Playwrightでは expect() がよく使われます。

Flow

ホテルサイトのサンプルコード

from playwright.sync_api import Page
import pytest

class TestHotelPlanisphere(object):

    # 前後の処理
    @pytest.fixture(scope="function", autouse=True)
    def page_fixture(self, page: Page):
        self.page = page
        yield
        self.page.close()

    def test_change_all_params(self):
        page = self.page
        page.goto("https://hotel-example-site.takeyaqa.dev/ja/reserve.html?plan-id=0", wait_until="networkidle")

        # 宿泊日を入力
        page.fill("#date", "2021/12/06")
        page.press("#date", "Tab")

        # 宿泊日数を入力
        page.fill("#term", "3")

        # 人数を入力
        page.fill("#head-count", "2")

        # お得なプランを選択
        page.check("#sightseeing")

        # お名前を入力
        page.fill("#username", "テスト太郎")

        # 確認のご連絡を選択
        page.select_option("#contact", "no")

        # 予約内容を確認するボタンをクリック
        page.click("#submit-button")

        # スクリーンショットの保存
        page.screenshot(path="スクリーンショットのパス")

基本の考え方

Playwrightのテストは基本的に、次の流れで記述します。

  1. 開く
  2. 入力する
  3. 操作する
  4. 確認する
