Desktop App Test

WinAppDriver

Windowsアプリを自動操作するためのテストツールです。Excel、メモ帳、電卓、社内ツールなどの操作確認に使います。

対象 Windowsアプリ
操作 クリック・入力・待機
テスト unittestで実行
使い始め 読み込み 基本構造 要素の取得 マウス操作 キーボード スクショ・待機 アサート・ログ

Structure

基本構造

unittest のルール

インポートしてクラスで指定します。下記の順番で自動的に実行されます。

  1. setUp() を呼ぶ：テストの準備
  2. test_...() を呼ぶ：実際のテスト
  3. tearDown() を呼ぶ：後片付け

Classの基本形

class クラス名:
    def メソッド名(self):
        処理

メソッドの最初の引数は、ほぼ必ず self です。

unittestでのクラスの書き方

class クラス名(unittest.TestCase):
    ...

setUp

def setUp(self):
    desired_caps = {}
    desired_caps["app"] = "アプリの指定"
    self.driver = webdriver.Remote(
        command_executor="http://localhost:4723",
        desired_capabilities=desired_caps
    )

アプリの指定

# Microsoft Store アプリ
desired_caps["app"] = "【アプリのAUMID】"

# PowerShellで調べる
Get-StartApps | Where-Object Name -like "*●●*"

# 社内ツール、メモ帳などデスクトップアプリ
desired_caps["app"] = r"【exeのフルパス】"

tearDown

def tearDown(self):
    # アプリケーションを閉じる
    self.driver.quit()

test_ 操作内容

def test_〇〇(self):
    # ① 操作
    要素を取得して操作する（click / send_keys など）

    # ② 結果を取得
    result = 結果表示の要素を取得

    # ③ 確認（assert）
    self.assertEqual(期待値, 実際の値)

Locator

要素の取得

2.0.0以降の書き方

検索対象 旧（非推奨・削除） 新（推奨）
Name find_element_by_name("...") find_element(AppiumBy.NAME, "...")
XPath find_element_by_xpath("...") find_element(AppiumBy.XPATH, "...")
ClassName find_element_by_class_name("...") find_element(AppiumBy.CLASS_NAME, "...")

Appium-Python-Client 1.3.0 の例

from appium.webdriver.common.mobileby import MobileBy

# Accessibility ID の場合
driver.find_element(MobileBy.ACCESSIBILITY_ID, "...")

# Name の場合
driver.find_element(MobileBy.NAME, "...")

# XPath の場合
driver.find_element(MobileBy.XPATH, "...")

# Class Name の場合
driver.find_element(MobileBy.CLASS_NAME, "...")

識別方法の考え方

優先度 方法 例
1 AccessibilityId（いちばん安定） self.driver.find_element(MobileBy.ACCESSIBILITY_ID, "AutomationId").click()
2 Name（表示文字で探す） self.driver.find_element(MobileBy.NAME, "表示されている文字")
3 XPath（最後の手段） self.driver.find_element(MobileBy.XPATH, "//Button[@Name='2']").click()

AutomationIdはInspectで確認します。

Screenshot / Wait

スクリーンショットと待機

スクリーンショットを撮る

import os
from datetime import datetime

def take_screenshot(self, label):
    daytext = datetime.now().strftime("%Y%m%d")
    time_text = datetime.now().strftime("%H%M%S")
    folder = r"C:\Users\yui.fukushima\Documents"
    folder_path = os.path.join(folder, f"●●_{daytext}")
    os.makedirs(folder_path, exist_ok=True)
    file_path = os.path.join(folder_path, f"{time_text}_{label}.png")
    self.driver.save_screenshot(file_path)

使い方

self.take_screenshot("before")
# 操作
self.take_screenshot("after")

ラベルは before や after など、自分で付ける名前です。

待機が必要なタイミング

  • アプリ起動直後
  • 画面切り替え直後
  • メニューを開いた直後
  • 計算結果の反映直後

固定待ち

time.sleep(1)
time.sleep(1.5)

1 は1秒、1.5 は1.5秒です。

要素が見つかるまで待つ

element = None
timeout = 10
poll_frequency = 0.5
end_time = time.time() + timeout

while time.time() < end_time:
    try:
        element = self.driver.find_element_by_識別方法("識別値")
        break
    except Exception:
        time.sleep(poll_frequency)

self.assertIsNotNone(element, "最大待機時間内に要素が見つかること")

テキストが期待値になるまで待つ

actual_text = ""
timeout = 10
poll_frequency = 0.5
end_time = time.time() + timeout

while time.time() < end_time:
    element = self.driver.find_element_by_識別方法("テキスト確認対象の識別値")
    actual_text = str(element.text)
    if actual_text == "期待するテキスト（完全一致）":
        break
    time.sleep(poll_frequency)

self.assertEqual("期待するテキスト（完全一致）", actual_text, "指定時間内に期待テキストになること")

WebDriverWait：要素が出るまで待つ

element = WebDriverWait(self.driver, 最大待機秒).until(
    lambda driver: driver.find_element_by_識別方法("識別値")
)

WebDriverWait：完全一致

WebDriverWait(self.driver, 最大待機秒).until(
    lambda driver: driver.find_element_by_識別方法("テキスト確認対象の識別値").text == "期待するテキスト"
)

WebDriverWait：部分一致

WebDriverWait(self.driver, 最大待機秒).until(
    lambda driver: "期待する部分文字列" in driver.find_element_by_識別方法("テキスト確認対象の識別値").text
)

Assert / Log

アサートとログ表示

含まれているか確認（部分一致）

element = self.driver.find_element_by_識別方法("識別値")
self.assertIn("期待する文字列", str(element.text), "失敗時のメッセージ")

True / False の確認

self.assertTrue(条件式, "失敗時メッセージ")
self.assertFalse(条件式, "失敗時メッセージ")

要素が存在することを確認

self.assertIsNotNone(element, "要素が取得できること")

実際の値を取得

element = self.driver.find_element_by_識別方法("識別値")
actual = str(element.text)
