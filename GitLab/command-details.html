コマンド詳細

ファイル確認系とGit系のコマンドを、実行例とログの見方まで少し詳しく整理します。

ファイル確認系コマンド

ファイル一覧を表示

script:
  - dir

実行しているフォルダ内のファイル一覧が表示されます。

さらに、特定のファイルが存在するか確認できます。

script:
  - dir README.md

dirの後ろにファイル名を書きます。

script:
  - ls
script:
  - ls README.md

dirとlsは、どちらもファイル一覧を表示するためのコマンドです。主に使う環境が違います。

コマンド 主な環境 役割
dir Windows ファイル一覧表示
ls Linux ファイル一覧表示

現在のフォルダを表示

script:
  - pwd

ジョブが動いている瞬間の作業フォルダを、ログで確認するためのコマンドです。

Git系

git status

script:
  - git status

ファイルがGitLabにPushされているか、つまりGitの現在状態を確認します。

ログ結果例

$ git status
HEAD detached at e923ce1
nothing to commit, working tree clean
  • HEAD detached at e923ce1: コミットe923ce1の状態で実行中。
  • nothing to commit: コミットする変更なし。
  • working tree clean: 変更されたファイルなし。

git log --oneline

script:
  - git log --oneline

どのコミットで実行されたか、つまりコミット履歴を確認します。
