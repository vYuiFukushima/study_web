yml書き方

GitLab CI/CDで実行する内容は、プロジェクト直下の.gitlab-ci.ymlに書きます。

ファイル名

ファイル名は必ず.gitlab-ci.ymlにします。

新たに作るときは、先にGitLab側でプロジェクトを新規作成します。

最小構成

stages:
  - test

echo_hello_world_test:
  stage: test
  script:
    - echo "Hello,World!"

日本語にすると

  • testステージ、つまり工程を作る。
  • echo_hello_world_testという仕事を作る。
  • その仕事はtestステージで実行する。
  • 仕事の内容としてecho "Hello,World!"を実行する。

ステージ

ステージは「処理のグループ分け」や「工程」のことです。

  • build プログラム作成
  • test テスト実行
  • deploy サーバーへ配置

基本の型

stages:
  - 工程名

テスト名:
  stage: 工程名
  script:
    - 実行内容

Pythonの実行

JSONをExcelに変換して出力するテストの例です。imageには、このJobをどんなパソコン環境で実行するかを書きます。

image: python:3.11

stages:
  - convert

xlsx_convert_test:
  stage: convert
  script:
    - pip install openpyxl
    - python 02_xlsx_converter.py
    - echo "変換終了"
  artifacts:
    paths:
      - "*.xlsx"

ホテルサイトの自動テスト

image: python:3.11

stages:
  - hotel

hotel_test:
  stage: hotel
  script:
    - python -m pip install playwright
    - python -m pip install pytest
    - python -m pip install pytest-playwright
    - playwright install chromium
    - playwright install --with-deps

    - pytest hotel_test.py -v
    - echo "hotel_test終了"

    - pytest hotel_test2.py -v -s
    - echo "hotel_test2終了"
  artifacts:
    paths:
      - "*.png"
