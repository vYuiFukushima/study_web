定期実行

Pipeline schedulesを使うと、指定した時間にパイプラインを自動実行できます。定期的な処理や日次バッチに使います。

設定手順

  1. プロジェクトを開き、ビルドからパイプラインスケジュールを開く。
  2. 新しいスケジュールをクリックする。
  3. 説明欄を入力する。
  4. タイムゾーンをOsakaにする。
  5. 時間間隔のパターンを選択する。カスタムの場合はCron構文で指定する。
  6. ターゲットブランチまたはタグを選択する。基本はmainを選ぶ。
  7. パイプラインスケジュールを作成をクリックする。

作成後、画面への反映まで少し時間がかかることがあります。

Runnerの確認

アサインされたプロジェクトのRunnerに、定期実行で使いたいRunnerがいることを確認します。

  1. 設定を開く。
  2. CI/CDを開く。
  3. Runnerを開く。
  4. アサインされたプロジェクトのRunnerを確認する。
  5. いない場合は、その他の利用可能なプロジェクトRunnerからプロジェクトへ割り当てを押す。

Cron構文

Cronは5つの値で実行する時間を指定します。

* * * * *
| | | | |
| | | | +-- 曜日（0-6：日-土）
| | | +---- 月（1-12）
| | +------ 日（1-31）
| +-------- 時（0-23）
+---------- 分（0-59）

よくある例

タイミング Cron
毎時ちょうど 0 * * * *
毎日0時 0 0 * * *
毎週日曜0時 0 0 * * 0
毎月1日0時 0 0 1 * *
毎年1月1日0時 0 0 1 1 *

書き方例

5 13 * * *      # 毎日13:05
30 8 * * *      # 毎日8:30
0 9 * * *       # 毎日9:00
10 9 * * 1-5    # 月-金 9:10
15 10 * * 1     # 月曜 10:15
*/2 * * * *     # 2分毎
25 * * * *      # 毎時25分
0 19 * * *      # 19:00

GitLabはCronの解析にfugitを使うため、1#2（第2月曜日）やsun%2（隔週日曜日）のような拡張構文も使えます。
