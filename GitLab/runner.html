自分のGitLab Runnerを使う

Windows上のGitLab Runnerをプロジェクトへ登録し、tagsで対象Runnerを指定して実行します。

GitLabにRunnerを登録する

  1. GitLabのプロジェクトページを開き、設定からCI/CDを押す。
  2. RunnerでプロジェクトRunnerを作成を選択する。
  3. タグ欄にwindowsと入力し、Runnerを作成を押す。
  4. プラットフォームのオペレーティングシステムをWindowsにする。
  5. Runner認証トークンをコピーし、メモ帳などに一時保管する。

タグはRunnerを選ぶための目印です。Runner側のタグと.gitlab-ci.yml側のtagsは一致させます。登録が完了したら、一時保管したトークンは削除します。

PowerShell登録コマンド

PowerShellを管理者として開き、GitLab Runnerのフォルダへ移動して登録します。

cd C:\GitLab-Runner

.\gitlab-runner.exe register

質問への回答

  1. GitLabのURLにはhttps://gitlab.comを入力する。
  2. 登録トークンには、GitLab画面で取得したRunner認証トークンを入力する。
  3. ランナー名には任意の名前を入力する。例: windows-local-runner
  4. executorにはshellを入力する。

executorは、ジョブをどの方式で実行するかを指定する項目です。

config.tomlを確認する

Runner登録後、次のファイルが作成されていることを確認します。

C:\GitLab-Runner\config.toml
  1. Windowsのスタートメニューで「メモ帳」と検索する。
  2. メモ帳を右クリックし、管理者として実行する。
  3. メモ帳のファイルから開くを選び、C:\GitLab-Runnerへ移動する。
  4. config.tomlを開く。
  5. shell = "pwsh"をshell = "powershell"へ変更し、上書き保存する。

ファイルが表示されない場合は、右下のファイル種類を「テキスト文書」から「すべてのファイル」に変更します。

Runnerを起動する

PowerShellを管理者として開き、GitLab Runnerのフォルダへ移動してサービスを開始します。

cd C:\GitLab-Runner

.\gitlab-runner.exe start

.\gitlab-runner.exe status

.gitlab-ci.ymlの書き方

tagsにGitLab側で設定したタグを指定します。

stages:
  - dentaku

dentaku_test:
  stage: dentaku
  tags:
    - windows
  script:
    - '& "C:\Users\yui.fukushima\AppData\Local\Programs\Python\Python311\python.exe" dentaku_0424.py'
    - echo "電卓テスト終了"

GitLab RunnerがLocalSystemとして動いている場合、Pythonの場所を知らないことがあります。そのためPythonをフルパスで指定します。
