GitLabとRunner

GitLabはソースコードやCI/CDを管理し、RunnerはGitLabから指示された処理を実際に実行します。

GitLabとは

GitLabは、ソースコードを管理するためのサービスです。

  • Gitリポジトリ管理
  • ソースコードの共有
  • Merge Requestによるコードレビュー
  • Issue管理
  • CI/CDによる自動実行

開発者はGitLabへソースコードをPushし、チームで管理・開発を行います。

GitLab Runnerとは

GitLab Runnerは、GitLabから指示を受けて実際に処理を実行するプログラムです。

  • Pythonスクリプト実行
  • テスト実行
  • Playwright実行
  • WinAppDriver実行
  • アプリのビルド
  • レポート作成
  • Slack通知

GitLabが「何を実行するか」を管理し、GitLab Runnerが「実際に実行する役割」を担当します。

GitLab.com共有Runner

GitLab.comがあらかじめ用意しているRunnerです。Runnerを自分で準備しなくても使用できます。

  • GitLab.comが管理する
  • 主にLinux環境で実行する
  • Dockerを利用して実行する
  • セットアップ不要ですぐ使える
  • PlaywrightなどのWebテストに利用できる

実行ログ例

実行ログの上部に次のような記載があれば、GitLab.com共有Runnerを使用しています。

on blue-6.saas-linux-small-amd64.runners-manager.gitlab.com
Preparing the "docker+machine" executor
Using Docker executor with image python:3.11

自分のGitLab Runner

利用者自身がインストールして管理するRunnerです。Windows専用ツールなど、共有Runnerでは実行しにくい処理で使います。

C:\GitLab-Runner
  • 利用者が管理する
  • Windows環境で実行できる
  • PowerShellで実行できる
  • WinAppDriverなどWindows専用ツールを利用できる
  • 実行環境を自由にカスタマイズできる

設定例

executor = "shell"
shell = "powershell"

使い分け

基本はGitLab.com共有Runnerを利用します。環境構築が不要で、PythonやPlaywrightの実行をすぐ始められるためです。

WinAppDriver、Windowsデスクトップアプリのテスト、社内専用ツール、特定のWindows環境が必要な処理では、自分のGitLab Runnerを利用します。

まとめ

項目 GitLab.com共有Runner 自分のGitLab Runner
管理者 GitLab 利用者
OS 主にLinux 任意、今回はWindows
環境構築 不要 必要
Playwright 使える 使える
WinAppDriver 使えない 使える
利用開始まで すぐ利用可能 セットアップが必要

基本的にはGitLab.com共有Runnerを利用し、Windows環境やWinAppDriverなど特殊な要件がある場合のみ自分のGitLab Runnerを利用します。
