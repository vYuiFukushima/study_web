実行方法

ファイルをGit管理対象に追加し、コミットしてPushするとGitLab側でパイプラインを確認できます。

ファイルの追加

  1. ディレクトリーを移動する。
  2. Git管理対象に追加する。
  3. 状態を確認する。
  4. コミットする。
  5. Pushする。

パスとコミットメッセージからコマンドを作成

cd <ディレクトリーのパス>

git add .
git status
git commit -m "任意のコミットメッセージ"
git push

(WinAppDriverを実行する場合は、WinAppDriver.exeの起動が必要)

確認するときの表示例

git statusで、コミット対象として追加されたファイルを確認できます。

Changes to be committed:
  new file: 02_xlsx_converter.py
  new file: report 1.json

パイプラインの実行

  1. VS Codeで新しいターミナルを開く。
  2. ディレクトリーを移動する。
  3. .gitlab-ci.ymlを追加、コミット、Pushする。
  4. GitLabで確認する。

パスからコマンドを作成

cd <ディレクトリーのパス>

git add .gitlab-ci.yml
git commit -m "Add first GitLab pipeline"
git push

リトライ実行

ビルドからパイプラインを開き、新しいパイプラインを実行します。成功または失敗ボタンから再実行ボタンを使うこともできます。
