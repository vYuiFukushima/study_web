CircleCI Pipelines

Pipelines画面

Pipelineを手動で実行する機能。

Pipelinesとは

CircleCIで実行されたPipeline（パイプライン）の履歴を確認する画面。

Pipelineとは、CircleCIが実行した一連の処理の単位を指す。

例：

GitHubへPush
↓
CircleCI起動
↓
テスト実行
↓
結果表示

この一連の流れが1つのPipelineとなる。

Pipelines画面で確認できること

  • Pipelineの実行履歴
  • 実行結果（成功・失敗）
  • 実行対象プロジェクト
  • 実行対象ブランチ
  • 実行日時
  • 実行時間
  • 実行のきっかけ

Trigger Pipeline

Pipelineを手動で実行する機能。

通常はGitHubへのPushなどで自動実行されるが、

Trigger Pipeline

を利用すると任意のタイミングで実行できる。

利用例：

  • 再実行したい
  • 特定ブランチで実行したい
  • Pushせずに実行したい

Pipeline一覧

Status

Pipelineの実行結果。

例：

Success
Failed
Running

Pipeline

Pipeline番号やコミット情報。

例：

#3017
Commit bce7810

Project

実行対象プロジェクト名。

例：

circleci-practice

Branch

実行対象ブランチ。

例：

main
develop
feature/login

Trigger

Pipelineが実行されたきっかけ。

例：

GitHub Push
Pull Request
Manual Trigger

Time

実行日時または実行時間。

例：

16m ago

↓

16分前に実行

例：

41s

↓

実行時間41秒

Share feedback

CircleCIへ意見や要望を送信する機能。

Pipeline表示フィルター

Pipeline一覧の表示対象を絞り込む機能。

All followed projects

フォローしているすべてのプロジェクトのPipelineを表示する。

My pipelines

自分が関係したPipelineのみ表示する。

Default branch

デフォルトブランチ（main など）のPipelineのみ表示する。

Pull requests

Pull Requestに関連するPipelineのみ表示する。

Pull Requestとは

変更内容をレビューしてもらい、対象ブランチへ取り込むための申請。

Pipeline検索フィルター

Pipeline一覧を条件で絞り込む機能。

Project

対象プロジェクトで絞り込む。

例：

circleci-practice
circleci-practice2

is any of

指定したいずれかの条件に一致するものを表示する。

Created

作成日時で絞り込む。

after

指定日時以降を対象とする。

Last 2 weeks

過去2週間以内に作成されたPipelineを表示する。

Add filter

新しい検索条件を追加する。
