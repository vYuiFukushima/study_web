Organization Settings画面

Organization Settingsとは

CircleCIの組織（Organization）全体の設定を管理する画面。

組織情報やメンバー管理、セキュリティ設定、外部サービス連携などを行うことができる。

Organization Settings画面の役割

Organization Settings画面は、組織全体の管理・運用設定を行うための画面である。

主な用途：

  • 組織情報管理
  • ユーザー管理
  • 環境変数管理
  • GitHub連携管理
  • セキュリティ管理
  • Slack連携
  • Orb管理
  • Runner管理
  • 監視設定

Organization Settingsとは

CircleCIの組織（Organization）全体の設定を管理する画面。

組織情報やメンバー管理、セキュリティ設定、外部サービス連携などを行うことができる。

Overview

組織の基本情報を確認する画面。

確認できる内容：

  • Organization ID
  • Organization Slug
  • Organization Name
  • Technical Contacts

Organization ID

組織を一意に識別するためのID。

(例)a1b2c3d4-e5f6-7890-abcd-1234567890ef

利用例：

  • CircleCI API利用時
  • サポート問い合わせ時
  • 組織識別

Organization Slug

CircleCI内部で利用される組織識別名。

github/sample-team

構成：

接続サービス名/組織名

Organization Name

組織名。

sample-team

Manage Technical Contacts

運用上の問題やサービス障害時の通知先を管理する。

People

組織に所属するユーザーを管理する画面。

できること：

  • メンバー確認
  • メンバー招待
  • 権限管理

Contexts

環境変数を管理する画面。

複数プロジェクトで共通利用する値を管理できる。

例：

API_KEY
TOKEN
PASSWORD

Contextの用途

機密情報をソースコードへ直接記載せずに管理する。

VCS Connections

バージョン管理システムとの接続設定。

VCS = Version Control System

対応例：

  • GitHub
  • GitLab
  • Bitbucket

Security

組織全体のセキュリティ設定を管理する画面。

例：

  • アクセス制御
  • 認証設定
  • セキュリティポリシー

Slack Integration

Slackとの連携設定。

利用例：

テスト成功通知
テスト失敗通知
デプロイ通知

Policies

組織全体のルールを管理する画面。

例：

特定設定の必須化
利用ルールの統一

Advanced

高度な組織設定を管理する画面。

通常利用ではあまり変更しない設定が含まれる。

Orbs

CircleCIのOrbを管理する画面。

Orbとは

CircleCIで利用できる再利用可能な処理セット。

例：

Slack通知
AWSデプロイ
Docker操作

Orbのメリット

同じ処理を毎回記述せずに利用できる。

Self-Hosted Runners

自社管理サーバーでジョブを実行するための設定。

Runnerとは

CircleCIジョブを実行する実行環境。

Self-Hosted Runnerの利用例

社内ネットワーク限定環境
オンプレミス環境

でジョブを実行したい場合。

Deploys

デプロイ関連の管理画面。

デプロイとは

アプリケーションを実行環境へ公開すること。

例：

開発環境へ公開
本番環境へ公開

OpenTelemetry Configuration

OpenTelemetryとの連携設定。

OpenTelemetryとは

システム監視や分析を行うための仕組み。

収集例：

  • ジョブ実行時間
  • エラー情報
  • パフォーマンス情報
  • ログ情報

OpenTelemetryの目的

システムの状態を可視化し、問題調査や性能分析を行う。
