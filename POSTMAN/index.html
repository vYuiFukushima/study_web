Postman

Postmanとは

Postmanとは、APIテストツールの一種である。

APIを実行するだけでなく、返ってきた結果を自動で確認することもできる。

1.パラメータ

Scriptsを書かなくても200 OKは表示されていた。APIが少ない時は人間の目で手動で確認できるけど、
APIが100個200個となると大変になる
⇒人間が見なくても自動で確認できるようにするためにScriptsを書く。

2.スクリプト

const jsonData = pm.response.json();

    pm.test("emailチェック", function () {
        pm.expect(jsonData.json.email).to.equal("test@example.com");
    });
    pm.test("nameチェック", function () {
        pm.expect(jsonData.json.name).to.equal("test_taro");
    });

pm.test(...)の中に入れると、Postmanは「これはテストです」と認識できる
pm.expect(実際の値).to.equal(期待値)

equal = 数字・文字・null

eql = {} のようなオブジェクト

const jsonData = pm.response.json();

pm.test("data_check",function(){
    pm.expect(jsonData.data).to.eql({});
});
pm.test("null_check",function(){
    pm.expect(jsonData.json).to.equal(null);
});

Basic Syntax

基本構文

  • Postmanのテストスクリプトは下記の言語で記述可能
    • JavaScript(主言語)
    • Chai(アサーションライブラリ)
  • 基本構文は下記
pm.test('テストの名称',function(){
    //テスト内容の詳細
    pm.expect("実行結果").to.be("期待結果")
});

JavaScript

テストを実行する処理を書く場所

pm.test('テストの名称',function(){

});

Chai

実行結果と期待結果が同じか確認する

pm.==expect==("実行結果").to.be("期待結果")

= 実際の値は期待値のはず！

基礎編

  • ステータスコードが200(OK)であることの確認
pm.test("ステータスコードチェック", function () { 
    //テストの名称　任意の文字列に変更可能
    pm.response.to.have.status(200);
    //()内が期待値　こちらも任意の値に変更可能
});
  • レスポンスボディに特定の文字列が含まれているかどうかの確認
pm.test("文字列部分一致チェック", function () { 
    pm.expect(pm.response.text()).to.include("期待値");
    //pm.response.text()
    // 　:レスポンスボディをそのままテキストデータに変換したもの。
    //    jsonなどの構造は一切引き継がれないので非推奨
});`

発展編

  • レスポンスボディの特定の要素が期待値と一致していることの確認
// レスポンスボディ期待値(仕様書)
// {
//     "success" : false,
//     "message" : "Some Error was Found."
// }

const json = pm.response.json();
//リクエストボディをjsonなどの構造を維持したまま取得する記述
const errorMessage  = "Some Error was Found.";

pm.test("エラーメッセージが仕様通りであることを確認する",function(){
    pm.expect(json.message).is.eql(errorMessage);
})
  • 真偽値(true/false)一致確認
// レスポンスボディ期待値(仕様書)
// {
//     "success" : false,
//     "message" : "Some Error was Found."
// }

const json = pm.response.json();

pm.test("success要素の値が仕様通りであることを確認する",function(){
    pm.expect(json.success).to.be.false;
    //[!]エラーになる記載例
    //  pm.expect(json.success).is.eql.false;
    //  pm.expect(json.success).to.be(false);
})
03

JavaScript / Python

JavaScriptとPythonの違い

変数の型を明記する必要がある

Python

# Python：左辺は変数名だけでよい
x = 0
y = 'hello'
Z = true

JavaScript

//JavaScript：**変数名と同時に変数の型を宣言しなくてはいけない**
//下記はエラー「ReferenceError: x is not defined」になる
x = 1;
//下記は通る(それぞれの型の違いについては後述)
const x = 1;
let y = 'Hello';
var Z = true;
//**一度宣言した変数は変数名だけで呼び出せる**
y = 'Goodbye';

関数の定義の仕方

Python

# python
def greet(name):
    print(f"{name}さん、こんにちは！")

JavaScript

//JavaScript
//基本構成は似ているが、下記はPythonにはない規則
// - {}で囲む必要がある
// - functionで明示的に関数であることを宣言する必要がある
function greet(name){
    console.log(`${name}さん、こんにちは！`);
}

Chaiのアサーションについて

  • PythonやJavaScriptの比較演算子とは記法が違う
    • </>/==/!=などは使わない
Chaiでの記述例 内容 備考
(A).to.equal(B) A == B
(A).to.be.a(データ型) Aが(データ型)である Array/Object/String/Number
(A).to.be.above(B) A > B A == Bはエラー扱い
(A).to.be.below(B) A < B A == Bはエラー扱い

Postmanのテスト関数の記法について

  • 基本の構文は下記
    • 少々構造が複雑でsyntaxErrorが出やすいので注意
pm.test('テストケース名',function(){
    //ここで実測値に対する処理を行うこともある
    // ex:list各要素のパラメータをforで判定するなど
    pm.expect('実測値').to.equal(`期待値`);
});

余談：JavaScriptの変数の型について

  • 再代入/再宣言を軸に3つの変数の型が定義されている
型 再代入 同一スコープでの再宣言 備考
const 不可 不可 基本はこれを使用
let 可 不可 値の変更・分岐が発生する場合に使用
var 可 可 非推奨(古い書き方)
  • 再代入
    • 変数の値を書き換えること
      これができない場合は厳密には変数ではなく定数となる
let string = `goodMorning`;
//(略)
string = 'hello';//これが変数stringの再代入
  • 再宣言
    • 変数を再度宣言すること
      変数名がごちゃごちゃになるので非推奨
var string = `goodMorning`;
//(略)
var string = 'hello';//これが変数stringの再宣言

該当する内容がありません。
