APIとは

APIの基礎知識

01

この章の目標

  • APIとは何かを説明できるようになる
02

APIとは

  • API = Application Programming Interface
  • 日本語で簡単に言うと 「ソフトウェア同士がやり取りするための窓口」

例えば、

天気アプリ
↓
API
↓
天気情報サービス

のように、ソフトウェア同士が情報をやり取りするときに利用される。

03

APIの種類

APIには様々な種類がある。

自動テスト業務では、主に Web API をテスト対象とすることが多い。

名称 詳細
Web API Webサービスを利用するためのAPI。例：Google API
ネイティブAPI OS（Windows、Linuxなど）が提供するAPI
ライブラリAPI 開発用ライブラリが提供するAPI。例：Java Spring Framework
ランタイムAPI プログラム実行環境が提供するAPI。例：Java SE API
データベースAPI データベースへアクセスするためのAPI。例：ODBC API
04

Web APIとは

  • インターネット経由でWebサービスを利用するためのAPI
  • HTTP または HTTPS 通信を利用してデータのやり取りを行う
05

HTTP / HTTPSとは

Webブラウザやアプリがサーバーと通信するためのルール。

例：

アプリ
↓
HTTP/HTTPS通信
↓
サーバー
06

身近な利用例

Google Mapの埋め込み

店舗のホームページにGoogle Mapが表示されている場合、

店舗側が地図機能を自作しているわけではなく、

店舗サイト
↓
Google Map API
↓
Google Map

を利用している。

SNSアカウントでログイン

例：

Googleアカウントでログイン
X（旧Twitter）でログイン
Facebookでログイン

Amazonアカウントで決済

オンラインショップで

Amazon Pay

を利用する場合もWeb APIが利用されている。

07

Web APIに対するテストの目的

Web APIは様々なシステムから利用されるため、品質を確認することが重要である。

概要 詳細
機能の検証 想定どおりに動作することを確認し、予期しないエラーや不具合を防止する
セキュリティの確保 脆弱性を発見・改善し、不正アクセスや情報漏洩を防止する
信頼性の向上 安定した動作を確認し、利用者や他システムからの信頼性を高める
パフォーマンスの測定 大量アクセスや高負荷状態でも正常に動作するか確認する
インターフェース契約の確認 システム間で決められた通信ルールが守られているか確認する
アップデートの影響評価 新機能追加や改修によって既存機能に影響が出ていないか確認する
08

補足：インターフェース契約とは

システム同士が通信する際の約束事。

例：

このURLにアクセスする
↓
この形式のデータを送る
↓
この形式のデータが返る

というルールを決めておくこと。

APIテストでは、この約束事が守られているかを確認する。

09

まとめ

  • APIは ソフトウェア同士がやり取りするための窓口
  • 自動テストでは主に Web API を対象とする
  • Web APIは HTTP/HTTPS通信 を利用してデータをやり取りする
  • Google MapやSNSログインなど、身近なサービスでも利用されている
  • APIテストでは、機能・セキュリティ・性能・互換性などを確認する
