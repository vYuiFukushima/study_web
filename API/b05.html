JSONについて

  • JSON = JavaScript Object Notation の略称
  • 元々はJavaScript向けのデータ形式として作られた
  • 現在は Python をはじめ様々なプログラミング言語で利用されている
  • REST APIにおけるデータのやり取りで最もよく使われる形式

04

JSONの基本構造

  • 各値は key（識別子） と value（値） のペア
  • keyとvalueの区切りは :（コロン）
  • 項目同士の区切りは ,（カンマ）
  • オブジェクトは {} で囲む
  • 引用符は必ずダブルクォーテーション " を使用する

JSONの書き方ルール

1. オブジェクトは {} で囲む

JSON全体は通常 {} で囲む。

2. key（項目名）は必ずダブルクォーテーション " で囲む

JSONでは key を省略できない。

3. 文字列は必ずダブルクォーテーション " で囲む

シングルクォーテーション ' は使用しない。

4. key と value は :（コロン）で区切る

key : value

の形式で記述する。

5. 項目同士は ,（カンマ）で区切る

複数の項目がある場合はカンマで区切る。

6. 最後の項目の後ろにカンマを付けない

JSONでは末尾カンマは使用しない。

7. 配列は [] で囲む

複数の値を並べる場合に使用する。

8. 使用できる主なデータ型

  • 文字列（String）
  • 数値（Number）
  • 真偽値（Boolean）
  • null
  • オブジェクト（Object）
  • 配列（Array）

9. JSONにはコメントを書けない

以下のようなコメント記法は使用できない。

//
/* */
13

JSON作成時のチェックリスト

  • {} で囲まれている
  • key がダブルクォーテーションで囲まれている
  • 文字列がダブルクォーテーションで囲まれている
  • key と value が : で区切られている
  • 項目同士が , で区切られている
  • 最後の項目に余分なカンマがない
  • コメントを書いていない
