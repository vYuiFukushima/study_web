ワークフロー作成：自動実行の実装

記述方法：ワークフロー概要

ワークフロー概要 
•name : ワークフロー名

•on : 実行トリガー条件

•schedule : 定期実行

	•push : ブランチへのプッシュ時
	
	•branches : ブランチ名  
	省略した場合は全pushでトリガー

	•pull-request：プルリクエスト作成時

		•branches : ブランチ名  
		省略した場合は全pushでトリガー

記述方法：実行内容の指定

実行内容の指定 
•**jobs** : 実行するジョブの内容

	•**runs-on** : 実行環境のOS
	
	**•steps :** 実行ステップの内容

•**name** : 実行ステップの名称

		**•uses :**  
		*既存の actionsを使用する場合
		
		•**run**  :  
		*任意のコマンドを実行する場合(GitHub Actions の中で、コマンドを実行する欄)

演習①

演習① 
name: echo-helloWorld-test

on:
  push:
    branches: [ main ]
jobs:
  echo_hello_world_test:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout codes  # GitHub Actions画面に表示される処理名
        uses: actions/checkout@v5 # GitHub上のコードを、Actionsの仮想PCにダウンロード

      - name: echo-helloWorld-test
        run: echo "Hello,World!"*

コードのチェックアウトを実行する=GitHubに置いてあるファイルを、GitHub Actionsが実行できる場所にコピーする
