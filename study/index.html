CI/CD研修で整理したい内容

1

CI/CDの全体像

開発者がコードを変更してから、テスト・ビルド・デプロイまで進む一連の流れを整理します。

2

GitHub Actionsの基本

Workflow、Job、Step、Runner、Eventなど、設定ファイルを読むために必要な用語を自分の言葉で整理します。

3

YAMLによる自動化設定

最小構成、push時実行、Pythonテスト実行の欄を用意し、自分でWorkflowを書き残します。

4

理解確認メモ

自分で問いを作り、答えや曖昧な点を書き残して理解を確認します。

Flow

整理の流れ

  1. Memo 1

    CI/CDの目的を書く

    なぜ自動化するのか、手作業の何を減らすのかを自分の言葉で残します。

  2. Memo 2

    GitHub Actionsの構成要素を整理する

    Event、Workflow、Job、Step、Runnerの関係を自分なりにまとめます。

  3. Memo 3

    Workflowファイルを書き写す

    研修で見たYAMLを自分で書き、どのタイミングで何が実行されるかを確認します。

  4. Memo 4

    疑問点を残す

    理解が曖昧な用語や、あとで試したい設定を確認メモに書きます。

メモのコツ

きれいにまとめるよりも、「自分はどこまで理解したか」「どこがまだ曖昧か」が後から見える形にしておくと、復習しやすくなります。
